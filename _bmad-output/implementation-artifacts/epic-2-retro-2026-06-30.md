## What Went Well

1. **Contract-first API discipline.** Story 2.1's response shapes (`{data,total,page,limit}`, `{data}`, `{data:string[]}`, `price` as a real number via `ColumnNumericTransformer`) were honored *exactly* by every downstream story. The contract held.
2. **One SSR pattern, reused three times.** Story 2.2 built the repo's first server-side data layer from scratch (native `fetch` service, deliberately not the client-only axios instance). Stories 2.3 and 2.4 reused it wholesale — no re-invention.
3. **Lessons demonstrably carried forward** (the standout strength):
   - **404-vs-empty-vs-error discipline** — *caught* in 2.2's review (category page conflated backend failure with "empty category"); *applied from the start* in 2.3 and 2.4.
   - **WCAG contrast trap** — 2.2 hit `text-warm-white on bg-alert` (fails AA); 2.3 explicitly avoided it (`bg-sand` + `text-brown`, ~7:1).
   - **Next 16 `useSearchParams` Suspense build-breaker** — 2.4 designed around it preemptively (filters passed as props, not via the hook).
   - The per-story "Previous Story Intelligence" sections are doing real knowledge transfer.

## Challenges / Recurring Friction

1. **Stale architecture doc → repeated reconciliation against live code.** The same gaps were re-litigated in every backend-touching story and each time documented as an "intentional variance":
   - Non-existent global `/api` prefix (a *regression guardrail* in 2.1, re-flagged in 2.2/2.3/2.4)
   - PascalCase-vs-lowercase entity files; snake_case-vs-camelCase columns
   - The `(shop)/products/?category=` route sketch diverged from **four times** (`/categories/[name]`, `/products/[id]`, `/search`)
   - Root cause: the doc was never updated to as-built reality, so the same decision is made over and over.
2. **Input hardening happens in review, not before.** Edge Case Hunter found loose input handling in *every* story — LIKE-wildcard escaping, `minPrice>maxPrice`, `NaN` price coercion, malformed-200 bodies, leading-zero / INT-overflow ids. The happy path ships; hardening is reactive.
3. **Two seed datasets, no canonical one.** Wired `product.seed.ts` (Dresses/Tops/Blazer, 2 SVGs each) vs un-wired `oren-products.sql` (11 PNG products, mostly single-image). 2.3 had to go fully image-count-agnostic because which dataset is live is ambiguous.
4. **Scope expanded mid-story without re-planning.** 2.3 absorbed multiple user-requested Stitch passes (bento gallery overriding AC#2, glass/mesh visuals, "Complete the Look") as "approved overrides." After 2.4 was marked `done`, commit `f8368ff` **redesigned the entire search UI** (FilterSidebar + PriceRangeSlider, sidebar layout, swapping out the reviewed FilterSortPanel) — with **no story record, AC, or code review**.
5. **Unowned slow-burn deferral.** The VND price vs hardcoded "$150" trust-copy mismatch surfaced in 2.2 (deferred), resurfaced in 2.3 (deferred again) — small, recurring, unowned.
